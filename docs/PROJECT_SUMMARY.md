# Wednesday Frog Bot - Краткий обзор проекта 🐸

## Цель проекта

Wednesday Frog Bot — это Telegram-бот для автоматической генерации и отправки изображений жабы с мемами про среду. Бот работает в автоматическом режиме, отправляя изображения по расписанию, а также поддерживает ручную генерацию по запросу пользователей.

## Ключевые технологии

Проект построен на современном технологическом стеке:

- **Python 3.8+** с **Asyncio** — асинхронное программирование для высокой производительности
- **PostgreSQL** — основное хранилище данных (чаты, метрики, использование, модели)
- **Redis** — кэширование и очереди для асинхронных задач
- **Celery** — распределенная система выполнения фоновых задач и планирования
- **Docker** — контейнеризация для упрощения развертывания

## Ключевой функционал

### Генерация изображений
- Интеграция с Kandinsky API для генерации изображений
- Использование GigaChat для создания промптов
- Поддержка нескольких моделей с возможностью переключения в runtime
- Circuit breaker для защиты от перегрузки API
- Retry механизм с exponential backoff

### Планирование задач
- Автоматическая отправка изображений по расписанию (несколько временных слотов в день)
- Поддержка часовых поясов
- Предотвращение дублирования отправок
- Интеграция с Celery для фоновых задач

### Администрирование
- Управление чатами (добавление/удаление)
- Управление моделями (Kandinsky, GigaChat)
- Управление администраторами
- Настройка лимитов использования
- Мониторинг метрик и статуса системы

### Rate Limiting
- Per-user ограничения (5 минут между запросами)
- Global ограничения (10 запросов в минуту)
- Месячные квоты для ручной генерации (`/frog`)
- Отслеживание использования через PostgreSQL

## Сравнение: Старое vs Новое

### Хранение данных
- **Старое:** JSON-файлы для хранения чатов, метрик и настроек
- **Новое:** PostgreSQL с асинхронными операциями через `asyncpg`, транзакционная целостность данных

### Архитектура выполнения
- **Старое:** Синхронный код с блокирующими операциями
- **Новое:** Асинхронный код (Asyncio) + Celery для фоновых задач, параллельная обработка запросов

### Планирование
- **Старое:** Встроенный планировщик с проверкой каждые 30 секунд
- **Новое:** Celery с поддержкой распределенного планирования и более точным временем выполнения

### Масштабируемость
- **Старое:** Ограничения из-за JSON-файлов и синхронного кода
- **Новое:** Горизонтальное масштабирование через Celery workers, connection pooling для БД
