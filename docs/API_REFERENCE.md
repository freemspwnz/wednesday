# Справочник команд API — Wednesday Frog Bot

Полный справочник команд для Telegram-бота "Wednesday Frog Bot". Документ предназначен для конечных пользователей и администраторов.

---

## 1. Обзор и условные обозначения

### Как использовать команды

Все команды вводятся через символ `/` в чате с ботом или в группе, где бот добавлен.

**Примеры:**
- `/start` — запуск команды
- `/frog` — генерация изображения
- `/help` — справка

### Условные обозначения

В описании команд используются следующие обозначения:

- `<аргумент>` — **обязательный** аргумент (должен быть указан)
- `[аргумент]` — **необязательный** аргумент (можно не указывать)
- `|` — альтернатива (выбор одного из вариантов)

**Примеры:**
- `/add_chat <chat_id>` — команда требует указания `chat_id`
- `/log [count]` — команда может использоваться без аргументов или с аргументом `count`
- `/force_send <chat_id|all>` — можно указать конкретный `chat_id` или `all`

### 2.3. `/frog`

**Назначение:** Генерация изображения жабы по запросу пользователя.

**Синтаксис:**
```
/frog
```

**Описание:**
- Генерирует новое изображение жабы с помощью нейросети Kandinsky
- Использует промпты, сгенерированные через GigaChat (или статические, если GigaChat недоступен)
- Применяет случайный стиль из предустановленных
- Учитывается в месячном лимите ручных генераций
- При ошибке генерации отправляет случайное изображение из архива

**Ограничения:**
- **Rate limit (per-user):** минимум 5 минут между запросами от одного пользователя
- **Rate limit (глобальный):** максимум 10 запросов в минуту от всех пользователей
- **Месячный лимит:** количество ручных генераций ограничено порогом (по умолчанию 70, максимум 100)
- Администраторы **не ограничены** per-user rate limit, но глобальный лимит и месячный порог применяются

**Процесс генерации:**
1. Бот проверяет rate limits
2. Проверяет месячный лимит генераций
3. Отправляет сообщение "🐸 Генерирую жабу для вас..."
4. Ставит задачу в очередь Celery (`wednesday.send_frog_manual`)
5. Celery worker генерирует изображение через Kandinsky API
6. Celery worker отправляет изображение с подписью пользователю
7. Celery worker сохраняет изображение локально в архив

**Примечание:** Команда `/frog` теперь использует Celery для асинхронной обработки. Обработчик выполняет только валидацию (rate limits, месячный лимит) и постановку задачи в очередь, что обеспечивает быстрый ответ пользователю (завершается за миллисекунды). Генерация и отправка изображения выполняются асинхронно в Celery worker, что исключает блокировку Event Loop и улучшает масштабируемость системы.

**Пример использования:**
```
/frog
```

**Ответ бота:**
```
🐸 Генерирую жабу для вас... Это может занять несколько секунд.
```

Затем отправляется изображение с подписью, например:
```
🐸 It's Wednesday, my dudes!
```

**Ошибки и fallback:**
- Если генерация не удалась, бот отправляет дружелюбное сообщение и случайное изображение из архива
- Если лимит исчерпан, бот сообщает об этом и предлагает дождаться автоматической отправки

**Пример ответа при исчерпании лимита:**
```
🚫 Лимит ручных генераций на этот месяц исчерпан.
Использовано: 70/100. Доступ к /frog закрыт после 70.
Ожидайте автоматических отправок по средам.
```

**Пример ответа при rate limit:**
```
⏰ Повторная генерация доступна через 180с
```

## 3. Административные команды (Admin Commands)

Команды, доступные только администраторам бота. Для выполнения этих команд пользователь должен быть добавлен в список администраторов через команду `/mod` или указан в переменной окружения `ADMIN_CHAT_ID`.

```
/status
```

- Выводит статистику генераций и лимиты
- Показывает количество активных чатов
- Отображает метрики производительности (успешность генераций, среднее время, circuit breaker)

**Пример использования:**

```
/status
```

**Назначение:** Получить логи бота для диагностики.

```
/log [count]
```

**Параметры:**
- `count` (необязательный) — количество дней логов (1-10). По умолчанию: 1 (последний файл)

**Описание:**

**Пример использования:**

**Назначение:** Принудительная отправка изображения жабы в чат(ы).

**Синтаксис:**
```
/force_send [chat_id|all]
```

**Параметры:**
- Без аргументов — показывает список активных чатов
- `<chat_id>` — ID конкретного чата для отправки
- `all` — отправка во все активные чаты

- Учитывается в месячном лимите генераций (если используется новая генерация)
- Отправляет копию изображения главному администратору (если указан `ADMIN_CHAT_ID`)

**Пример использования:**

```
/force_send           # Показать список чатов
```

### 3.13. `/set_frog_limit`

**Пример использования:**
```

```

**Ответ бота:**
```
✅ Порог /frog установлен: 70 (текущее использование: 45/100)
```

**Необходимые права:** Администратор

- Устанавливает текущее количество ручных генераций `/frog` за текущий месяц

- Значение автоматически ограничивается месячной квотой (не может превысить её)

## 5. Взаимодействие в группах (Group Commands)

Бот может работать в группах и супергруппах Telegram. При добавлении бота в группу он автоматически добавляется в список чатов для рассылки.

### 5.1. Автоматическое добавление в группы

**Описание:**
- При добавлении бота в группу он автоматически регистрирует группу в базе данных
- Группа начинает получать автоматические рассылки по расписанию
- Бот отправляет приветственное сообщение при добавлении

**Приветственное сообщение:**
```
🐸 Привет! Я Wednesday Frog Bot.

Я присылаю картинки с жабой по средам (09:00, 12:00, 18:00 по Мск),
а также по команде /frog (если не превышен лимит ручных генераций).

Доступные команды:
• /start — информация
• /help — справка
• /frog — сгенерировать жабу сейчас
```

### 5.2. Использование команд в группах

**Синтаксис команд в группах:**
- Команды можно использовать как обычно: `/frog`, `/help`, `/start`
- Можно также использовать с упоминанием бота: `/frog@botname`, `/help@botname` (если у бота есть username)

**Примеры:**
```
/frog
/frog@wednesday_frog_bot
/help
/status
```

### 5.3. Автоматическое удаление из групп

**Описание:**
- При удалении бота из группы он автоматически удаляется из базы данных
- Группа перестаёт получать автоматические рассылки
- Никаких уведомлений не отправляется

### 5.4. Ограничения в группах

- Все команды работают так же, как в личных чатах
- Административные команды доступны только администраторам бота (не администраторам группы)
- Rate limits применяются глобально (не зависят от типа чата)

## 7. Дополнительная информация

### 7.1. Rate Limiting

Бот использует несколько уровней rate limiting для защиты от злоупотреблений:

1. **Per-user rate limit:**
   - Минимум 5 минут между запросами `/frog` от одного пользователя
   - Не применяется к администраторам

2. **Глобальный rate limit:**
   - Максимум 10 запросов `/frog` в минуту от всех пользователей
   - Применяется ко всем, включая администраторов

3. **Месячный лимит:**
   - Ограничение на количество ручных генераций за месяц
   - По умолчанию: порог 70, квота 100
   - Настраивается через `/set_frog_limit` и `/set_frog_used`

### 7.2. Автоматические рассылки

Бот автоматически отправляет изображения жабы:
- **Расписание:** Каждую среду
- **Времена отправки:** Настраиваются через переменную окружения `SCHEDULER_SEND_TIMES` (по умолчанию: 09:00, 12:00, 18:00 по Мск)
- **Часовой пояс:** Настраивается через `SCHEDULER_TZ` (по умолчанию: Europe/Amsterdam)

### 7.3. Обработка ошибок

При ошибках генерации бот:
1. Отправляет дружелюбное сообщение пользователю
2. Отправляет случайное изображение из архива (fallback)
3. Уведомляет всех администраторов о деталях ошибки
4. Логирует ошибку для диагностики

### 7.4. Получение ID чата или пользователя

Для получения ID чата или пользователя можно использовать:
- Бот [@userinfobot](https://t.me/userinfobot) — показывает ваш user_id
- Бот [@getidsbot](https://t.me/getidsbot) — показывает ID чата и пользователей
- В группах: переслать сообщение в [@RawDataBot](https://t.me/RawDataBot)

## 8. Часто задаваемые вопросы (FAQ)

### Q: Почему команда `/frog` не работает?

**A:** Возможные причины:
- Исчерпан месячный лимит генераций (используйте `/status` для проверки)
- Сработал rate limit (подождите 5 минут)
- Проблемы с API (проверьте `/status`)

### Q: Как добавить бота в группу?

**A:**
1. Откройте настройки группы
2. Выберите "Добавить участников"
3. Найдите бота по username или через поиск
4. Бот автоматически добавится в список чатов для рассылки

### Q: Как получить права администратора?

**A:** Обратитесь к существующему администратору. Он может добавить вас через команду `/mod <ваш_user_id>`.

### Q: Бот не отвечает на команды

**A:**
1. Проверьте, что бот запущен (используйте `/status` если вы админ)
2. Убедитесь, что вы используете правильное имя бота
3. Проверьте, не находится ли бот в режиме Support Bot (технические работы)

### Q: Как изменить расписание автоматических отправок?

**A:** Расписание настраивается через переменные окружения при запуске бота. Обратитесь к администратору сервера.
